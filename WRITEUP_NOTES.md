# Invincible: Bayer Leverkusen's Unbeaten 2023/24 Bundesliga Season

## Rationale And Design Decisions

This project asks a simple question: what does an unbeaten season look like when viewed both as a full narrative and as a set of individual match patterns? I chose Bayer Leverkusen's 2023/24 Bundesliga season because it provides a naturally compelling story, but it also contains enough event-level detail to support genuine exploration. Instead of building a general soccer dashboard, I focused on one narrative arc: how Leverkusen stayed unbeaten across all 34 league matches, and what passing patterns help explain that run.

The visualization is organized as an overview-to-detail system. The first view is a season-level cumulative-points chart. I chose a line chart because the unbeaten season is fundamentally a temporal story, and cumulative points communicate momentum more clearly than a table of results. I paired the line with a win/draw ribbon so the viewer can immediately see that there were no losses. I also added autoplay and a round scrubber so the season can be experienced as a guided progression rather than a static summary. This satisfies the animation requirement in a way that is tied directly to the data rather than added as decoration.

Below that overview, the interface shifts into match-level analysis. The pass map uses a soccer pitch because passing is inherently spatial, and a pitch preserves both location and direction. Arrows make movement and intent legible in a way that a table or a simple count chart would not. The player ranking chart provides a second, non-spatial view for comparison, while the selected passing-move replay gives details on demand by animating one possession sequence after the user clicks a pass. I also kept a timeline histogram directly under the pass map so viewers can connect where passes happened with when they happened in the match.

The main interactions are dynamic query filters, brushing, clicking, and autoplay. Users can filter by match, player, pass type, play situation, and pass result; brush the timeline to focus on a minute range; click bars to isolate a player; click season matches to load a fixture; and click a pass to animate the full sequence it belongs to. I added three MVP entry cards so that viewers who do not already know soccer tactics still have an easy starting point. This was important because one risk in soccer visualizations is that they become legible only to people who already understand the sport.

I considered several alternatives before settling on this design. A pass network was one option, but it would have hidden field position and made it harder to understand movement into dangerous areas. A pure season-results visualization would have been cleaner, but too shallow for the assignment because it would not have supported rich interactive exploration. A passing-only dashboard was also possible, but it felt less memorable and weaker as a narrative. The final design combines a strong season story with match-level evidence, which I think better satisfies both the exploration and storytelling goals of the assignment.

## Development Process

I began by examining the StatsBomb Open Data files for the 2023/24 Bundesliga season and confirming that Bayer Leverkusen's full unbeaten league campaign was available. I then wrote a preprocessing script to convert the raw event data into a lighter JSON file designed specifically for the web interface. That script extracts passes, derives features such as forward passes, entries into the attacking third, entries into the penalty area, and passing sequences, and also computes match-level summaries such as result, points, and cumulative points. This preprocessing step was important because it reduced the amount of work done in the browser and made linked interactions much easier to implement in D3.

After the data pipeline was working, I built the visualization as a static site in the `docs/` folder so it can be deployed directly through GitHub Pages. I implemented the season chart first, then linked it to the match explorer, and then added the pass map, ranking view, timeline brush, and sequence replay. A large part of the development process was iterative design rather than pure coding: rethinking the opening state, simplifying terminology for viewers who may not know soccer, and reorganizing the panels so the page had a clear visual hierarchy. The final result is much more focused than my earlier drafts because I repeatedly cut features that did not strengthen the main story.

Roughly, I would describe this project as about 20 to 25 people-hours of work. The most time-consuming parts were preprocessing the soccer event data, deciding which interactions were actually useful rather than just possible, and refining the layout so the interface felt intentional instead of looking like a generic dashboard. I did use an LLM to help accelerate parts of the implementation process, especially for D3 scaffolding, iteration on interface text, and small design revisions. I felt reasonably comfortable coding this way, but only when I kept checking the generated code against the actual dataset and against my own goals for the project. The LLM was most helpful for speed, but the harder parts of the assignment still required manual judgment: choosing the narrative scope, defining derived measures, and deciding which interactions improved understanding instead of adding clutter.
